### Findings

[WARN ‚Äî RESOLVED 2026-06-30] daydream/api/slots.py:147-157 (`delete_toon`) and :134-144 (`leave_session`) ‚Äî Bodyless state-changing POST endpoints are CSRF-able in the default `tailscale` access mode; the SameSite=Lax mitigation recorded for the slot endpoints does not apply in that mode, and `delete` makes the impact irreversible.
  **Resolution:** Fixed in this push by `CsrfOriginMiddleware` (`daydream/api/csrf.py`), registered in `daydream/server.py` ahead of `AccessMiddleware`. It rejects (403) any unsafe-method HTTP request whose `Origin` (or `Referer` fallback) netloc does not equal the request `Host`; requests with no `Origin`/`Referer` pass (preserving the non-browser `bin/game world swap` CLI, curl, and the test suite), and safe methods including the WS upgrade GET are never gated. The confused-deputy cross-site form POST now carries the attacker page's `Origin` and is refused before the route runs. Covered by `tests/test_csrf_middleware.py` (cross-origin reject, same-origin allow, no-origin allow, safe-method allow, pre-route reject, matcher unit). The friend-scope authorization dimension (any authed session may act on any slot) remains an accepted risk below; only the CSRF/confused-deputy dimension is closed here.
  Attack vector: A tailnet-resident player (a current daydream user) visits an attacker-controlled web page anywhere on the internet. The page auto-submits a top-level form POST to `http://<victim-tailnet-addr>:54321/api/slots/1/delete` (or `/kick`, or `/api/session/leave`). Because the request leaves the victim's own machine, its TCP source IP is a tailnet address, so `AccessMiddleware` passes it (`daydream/api/access.py:65-68`, source-IP-only gate). Because `is_authed()` returns True unconditionally in tailscale mode (`daydream/api/auth.py:93-95`), the session cookie is never consulted, so its `SameSite=Lax` attribute is irrelevant. Because the handlers read nothing from the request body, a "simple" cross-site form POST (no custom header, no CORS preflight) reaches them. Result: permanent toon deletion cascading to the toon's items + memories (`daydream/toons.py:324-339`), or a recoverable kick, or a forced leave, with no token and no same-origin check. Preconditions: the attacker needs the victim's tailnet IP / MagicDNS name and the port (54321, published in-repo), and the victim must load the page; the cross-origin response is opaque but the side effect lands.
  Why this is not merely the accepted CSRF risk: the prior SECURITY.md accepts "no CSRF token on the slot endpoints" on two stated grounds ‚Äî SameSite=Lax blocks the cookie cross-site, and `AccessMiddleware` 403s cross-site POSTs "from outside the tailnet." Both grounds fail for the confused-deputy scenario in the default mode: auth does not use the cookie, and the forged request's source IP is the victim's own tailnet IP, not "outside the tailnet." This turn also introduces `delete`, the first irreversible verb in the family, so the blast radius is now permanent data loss rather than a recoverable kick or a forced logout. `create` (`slots.py:64`) and `world swap` (`daydream/api/world.py:116`) call `await request.json()` and therefore have incidental friction (a form-encoded body fails to parse; a JSON `fetch` trips a preflight the server does not approve), so the cleanly exploitable set is `delete` / `kick` / `leave`.
  Evidence: `daydream/api/auth.py:93-95` (`is_authed` unconditionally True in tailscale mode); `daydream/api/access.py:65-68` (gates on `scope["client"][0]` only); `daydream/api/slots.py:128,147-157` (`kick`/`delete`/`leave` ignore the body); `daydream/server.py:42` (`https_only=False`, default SameSite=Lax cookie, never consulted in tailscale mode).
  Remediation: require a forgery-resistant signal on state-changing POSTs that a cross-site simple-form POST cannot supply. Options: a custom request header set by `main.js`'s `fetch()` calls and required server-side (any custom header forces a CORS preflight the server can leave unanswered); an `Origin` / `Referer` allowlist check in a small middleware; or a per-session CSRF token echoed in the request. Apply to `delete` / `kick` / `leave` at minimum (and `create` / `swap` for completeness). If the owner judges the victim's tailnet address sufficiently un-guessable within the friend group, this may instead be re-recorded as an accepted risk with the corrected rationale (the SameSite reasoning does not hold in tailscale mode).

### Accepted Risks

Carried forward from prior reviews; none re-flagged as findings. Note the friend-scope item is **extended this turn**: `POST /api/session/leave` and `POST /api/slots/{slot}/delete` join create/claim/kick (any authed session may act on any slot; `delete` additionally destroys the toon's items + memories permanently). SPEC 2026-06-29 scopes per-session ownership to v2 `multi-user-shared-world`, and `slots.py:151` documents it. The CSRF dimension of these same two new endpoints is the WARN above.

- v1 friend-scope on the slot endpoints (`daydream/api/slots.py`): any authed session can create / claim / kick / **leave / delete** any slot; no per-session ownership enforcement. v2 introduces the per-session ownership check.
- Tailscale-mode auth: `is_authed()` returns True unconditionally (`daydream/api/auth.py:93-95`) and `POST /api/login` short-circuits to authed without checking the password; `AccessMiddleware` has already CGNAT-gated the request. Documented in the module docstring + CLAUDE.md.
- Cookie `https_only=False` in `daydream/server.py:42` (friend-scope; LAN/Tailscale only).
- No CSRF token on `/api/login`, `/api/logout`. `SameSite=Lax` blocks the cookie on cross-site POST in **public** mode; in tailscale mode see the WARN above.
- `POST /api/world/swap` (`daydream/api/world.py`) is gated by `is_authed` only (friend-scope); the target is confined under `data_dir()` via `resolve()` and probed read-only+immutable before any live-state change. Same trust class as the slot endpoints.
- The `100.64.0.0/10` Tailscale CGNAT hardcoding in `daydream/api/access.py`. A self-hosted Headscale with a custom range would need it updated.
- `AccessMiddleware` reads `scope["client"][0]` directly rather than honoring forwarded-for headers. Secure default given uvicorn does not run with `--proxy-headers`. An operator behind a reverse proxy must add `--forwarded-allow-ips` AND ensure the proxy strips inbound `X-Forwarded-For`.
- `bin/game` sources `.env` and `~/.config/daydream/secrets.env` via `set -a; source`; a writable env file equals code execution. Operator-controlled, gitignored, standard practice. The `DAYDREAM_DRIFT_*` / `DAYDREAM_MEMORY_*` parsers float/int-coerce with a `ValueError` fallback to default and never reach security-sensitive paths.
- `bin/game cmd_logs` passes `$1` through `_log_file` without validation. Operator-invoked only.
- `/cache/{world}/{target_kind}/{target_id}/{filename}` in `daydream/server.py:120-132` is intentionally unauthenticated so pre-auth `<img>` fetches render; segment validation blocks traversal; `AccessMiddleware` is the real gate.
- `/status/drift` in `daydream/server.py:62` is unauthenticated by design (only `AccessMiddleware` gates it); returns three integer counters, no PII/secrets.
- Data-skill `set_mood` effect handler accepts an LLM-chosen `toon_id` without scoping it (`daydream/skills/effects.py`). v2's per-effect jsonschema + target-authorization pipeline is tracked under BACKLOG `skills-authoring-and-security`.
- Stored prompt-injection via captured memory text bypasses `wrap_player_input` containment (`daydream/memories.py`, `daydream/drift.py:224-230` template). Capture-time banlist + `<memory>` wrapping in place; `</memory>` substrings not neutralized at retrieval. v0 impact bounded to mild voice/tone deviation. Deeper fix is v2.
- `bin/vllm-bootstrap` / `bin/memory-bootstrap` interpolate `$MODEL` into an unquoted heredoc. Same operator-trust class as `bin/game` env sourcing.
- `_ensure_session_id` (`daydream/api/auth.py:29-39`) stamps a fresh UUID into the session in tailscale mode even without `/api/login`. Consistent with the tailnet-is-auth posture; the UUID is the slot picker's controller identity, not an auth token.
- Unbounded request body on `POST /api/slots/{slot}/create`. Same trust class as the documented friend-scope posture.
- `qpeek` / engine bootstrap scripts clone from `github.com/peterzat/*` and `pip install -e`. Same trust boundary as the dev toolchain.
