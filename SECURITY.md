### Accepted Risks

Carried forward from prior reviews; none re-flagged this scope:

- Cookie `https_only=False` in `daydream/server.py:42` (friend-scope; LAN/Tailscale only).
- No CSRF token on `/api/login` and `/api/logout`. `SameSite=Lax` blocks cross-site POST from carrying the session cookie; worst case under the documented threat model is forced-logout from a same-site phishing page.
- The `100.64.0.0/10` Tailscale CGNAT hardcoding in `daydream/api/access.py`. Self-hosted Headscale with a custom range would need this updated. Documented in CLAUDE.md.
- `AccessMiddleware` reads `scope["client"][0]` directly rather than honoring forwarded-for headers. Secure default given uvicorn does not start with `--proxy-headers`. Operators putting the app behind a reverse proxy must add `--forwarded-allow-ips` AND audit that the proxy strips inbound `X-Forwarded-For`.
- `bin/game` sources `.env` and `~/.config/daydream/secrets.env` via `set -a; source`; a writable env file is equivalent to arbiter code execution. Operator-controlled, gitignored, standard practice. The `DAYDREAM_DRIFT_*` (including `DAYDREAM_DRIFT_LLM_ENABLED`, `DAYDREAM_DRIFT_LLM_TOP_K`, `DAYDREAM_DRIFT_SUPPRESS_OCCUPIED`, `DAYDREAM_DRIFT_MOOD_DRIFT_ENABLED`, `DAYDREAM_DRIFT_MOOD_DRIFT_PROB`) and `DAYDREAM_MEMORY_*` env vars sit inside this same trust boundary; their parsers float-/int-coerce with a `ValueError` fallback to the default and never propagate to security-sensitive code paths.
- `bin/game cmd_logs` passes `$1` through `_log_file` without validation. Operator-invoked only; anything the operator can `tail` through this path they can already read directly.
- `bin/qpeek-bootstrap` clones from `github.com/peterzat/qpeek` and runs `pip install -e`. Same trust boundary as the rest of the dev toolchain.
- `/cache/{world}/{target_kind}/{target_id}/{filename}` in `daydream/server.py:118-130` is intentionally unauthenticated so pre-auth `<img src>` fetches render. `AccessMiddleware` is the real gate; segment validation blocks traversal.
- Tailscale-mode `POST /api/login` at `daydream/api/auth.py:32-34` short-circuits to `authed=True` without checking the password. `AccessMiddleware` has already CGNAT-gated the request. Documented in module docstring + CLAUDE.md.
- Data-skill `set_mood` effect handler accepts an LLM-chosen `toon_id` without scoping it (`daydream/skills/effects.py:128-151`). v2's per-effect jsonschema + target-authorization pipeline is tracked under BACKLOG `skills-authoring-and-security`. The `daydream.toons.set_mood` helper itself is a parameterized UPDATE; the unscoped-target risk lives at the effect-handler call site, not in the helper.
- `bin/vllm-bootstrap` and `bin/memory-bootstrap` interpolate `$MODEL` (sourced from `DAYDREAM_VLLM_MODEL` / `DAYDREAM_MEMORY_MODEL`) into an unquoted heredoc that becomes Python source. Same operator-trust class as `bin/game`'s env sourcing.
- Stored prompt-injection via captured memory text bypasses `wrap_player_input` containment (`daydream/skills/data.py:362`, `daydream/memories.py:175-209`, `skills/{rook,iris}.json:6`, plus the parallel drift instance at `daydream/drift.py:179-185`). Capture-time banlist is in place; `<memory>...</memory>` separator wraps the rendered text; `</memory>` substrings are NOT neutralized at retrieval time. Realistic impact at v0 is bounded to mild voice/tone deviation, not auth bypass / RCE / data exfiltration. Deeper fix is v2's `skills-authoring-and-security`.
- `/status/drift` in `daydream/server.py:60-80` is unauthenticated by design (no session check; only `AccessMiddleware` gates it). Returns three integer counters with no PII or secrets; in tailscale mode it is tailnet-only, in public mode it exposes "drift has fired N times" to any reader. Same trust class as `/login` and `/cache/...`.
