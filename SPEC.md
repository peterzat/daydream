## Spec — 2026-07-01 — playable quest loop: two-object verbs, object state, a new world

**Goal:** Turn the well-built object/verb/effect substrate into a first *complete* interactive loop — discover → gather → exchange → unlock → reveal — by activating the dormant two-object-verb plumbing, adding persistent object state, extending the world-authoring envelope to express stateful interactive objects, and authoring a brand-new canonical world (The Clockmaker's Loft) that hosts the quest. Shelves `bunny` as the active world. Exercises real gameplay and yields the deterministic golden-playthrough integrity test the architecture has been missing.

### Acceptance Criteria

- [ ] **The quest is completable end to end (golden playthrough).** A deterministic test loads `worlds/clockmakers-loft.json` into a temp DB (the `tests/test_scenario.py` / `tests/test_world_load.py` pattern) and drives the fixed sequence — `read` ledger → `take` gear → `give` gear to Tace → `use` case-key on clock-case → `open` clock-case — asserting each transition: the ledger's authored clue text surfaces; the gear moves into inventory; `give` reparents the gear onto Tace (present in exactly one container, not duplicated), shifts Tace's mood, and puts a `use`-able case-key in the player's inventory; `use` flips the clock-case `locked→unlocked`; `open` flips `unlocked→open`, emits the authored payoff, and spawns the reward exactly once. The deterministic path makes **zero LLM calls** (asserted by a spy or by construction).

- [ ] **Two-object verbs exist and are scope/kind-validated.** `give` (dobj `thing`, iobj `toon`) and `use` (dobj `thing`, iobj `thing`) join the closed verb set with `needs_iobj`; `execute_command` gains an iobj gate symmetric to the dobj gate — a missing, out-of-scope, or wrong-kind iobj (`give gear to <a thing>`, `use gear on <a toon>`) yields gentle narration and **no mutation**. `give` also refuses when the dobj is not carried and when the iobj is the actor itself (no give-to-self). Verifiable by `tier_short` unit tests (happy path + each refusal).

- [ ] **`open` is state-gated, `read` surfaces authored text, `examine` reflects state.** `open` on a `locked` object refuses with the object's authored locked-text and does not open; on `unlocked`/`closed` it transitions to `open`, emits authored open-text, and reveals its `contains` payload; on already-`open` it says so and does not re-spawn (spawn dedup). `use` with the wrong item or in the wrong from-state changes nothing and narrates a soft "nothing happens"/wrong-state line. `read` narrates a readable's authored `text` (distinct from `examine`'s physical description) and degrades gently when absent. `examine` appends the current-state description without overwriting the physical seed. `tier_short`-testable.

- [ ] **Object state persists through the effect API with no migration.** State lives as free-form `properties.state`; every transition flows through the existing `set_property` effect (no new mutation path, no schema/migration change). After `use`/`open`, re-reading the object via `objects.get` shows the new state, and a freshly built `state_snapshot` reflects it — i.e. the mutation is durable in the shared DB substrate, not connection-local. (Two-session visual confirmation is the one manual glance.)

- [ ] **The envelope can author stateful interactive objects, and spawns can carry verbs.** `bootstrap.load_world` accepts and persists — on items: an optional `properties` dict (`state`, `state_text`, `use` rule, `contains`, `locked_text`/`open_text`, …), an optional `verbs` list, readable `text`, and a `fixture` flag selecting an immovable prototype whose defaults exclude `take`/`drop`; on toons: a `properties` dict (`wants`/`gives`/`gives_text`). The `spawn_object` effect accepts an optional `verbs` list written to the spawned object's `properties.verbs` (so the given case-key is `use`-able). Cross-references resolve by name (no authored ids). Malformed new fields fail validation loudly. Verifiable by `tier_short`/`tier_medium` loader + effect unit tests (per `tests/test_world_load.py`, `tests/test_effects.py`).

- [ ] **Both producers drive the two-object verbs.** The parser grounds `"give X to Y"` / `"use X on Y"` / `"use X with Y"` to the verb + a dobj id + an iobj id (deterministic fast-path when both names are in scope; the existing LLM path otherwise), and the WS `{kind:"command", verb, dobj_id, iobj_id}` frame reaches `execute_command` with the iobj. The `verb_bar` snapshot payload carries `needs_iobj` (+ `valid_iobj_kinds`), and `web/assets/main.js` implements two-step click staging: staging a `needs_iobj` verb, clicking the dobj, then clicking a kind-valid iobj sends both ids; single-object verbs keep the one-click path. Verifiable by `tier_short` parser tests and a `tests/test_frontend.py`-style source-scan of the staging logic (no headless-browser dep, consistent with the project's frontend-test posture).

- [ ] **The new world is canonical and on-aesthetic, and the loop survives an LLM outage.** `worlds/clockmakers-loft.json` validates and loads: 5 rooms with bidirectional exits, 4 toons (1 human-claimable + 3 NPCs with WHIMSY-toned `dialogue`), the stateful clock-case (fixture) + repair-ledger (readable) + escapement gear, and 2 ambient room-skills. `bin/game world reset` targets the new world and the load-bearing CLAUDE.md canonical-world references point to it (`bunny.json` retained as the `test_scenario.py` loader fixture). With vLLM down, the deterministic verbs (`read`/`take`/`give`/`use`/`open`, all no-LLM) still complete the quest; only NL free-text parsing and `talk` degrade to the foggy message.

- [ ] **Tiers green; docs, version, and backlog rolled forward.** `bin/game test short` and `bin/game test medium` exit 0 with the new unit + playthrough tests (GPU/LLM-free via mocks; the one `verb_bar`-shape assertion in `tests/test_ws.py` updated for the additive fields). CLAUDE.md's closed-verb list and command-bus section document `give`/`use`/`open`/`read` and the `properties.state` model; `daydream/version.py:WORLD_VERSION` takes a MINOR bump; BACKLOG's `two-object-verbs` is closed and the per-object-verbs slice of `deep-prototype-inheritance-and-per-object-verbs` is noted as shipped.

**Key facts the implementer needs.**
- **The plumbing is dormant, not absent.** `VerbSpec.needs_iobj`, `Parse.iobj_id`, the WS command frame's `iobj_id`, and the parser's LLM grounding schema already carry an indirect object; nothing declares a verb that uses it. `VerbSpec` needs a new `valid_iobj_kinds` field.
- **No migration.** Object state is free-form `properties_json` mutated by the existing `set_property` effect. The effect vocabulary (`narrate`/`set_property`/`move_object`/`spawn_object`) already covers every quest beat; the only effect change is an optional `verbs` passthrough on `spawn_object`.
- **Generation policy (load-bearing).** Quest beats are **pre-baked** authored strings in the envelope (locked/open/give text, the ledger clue), per local-at-runtime / Opus-at-design-time. The local LLM only voices ambient `talk`/drift around the fixed beats. No cloud key anywhere; deterministic verbs make no LLM call.
- **Internal `world_id` stays `w-bunny`** (the loader hardcodes it); only the human-readable world name changes.
- **Test determinism.** `tests/conftest.py` disables drift and NPC memory by default, so the playthrough is deterministic. The `give→memory→talk` enrichment is a best-effort CPU-only runtime feature (fail-closed, off in tests) — assert the visible mood change + spawned key, not the memory row.

**Tier placement.** Unit tests for verbs/handlers/iobj-gate/parser-fast-path/loader-passthrough/spawn-verbs → `tier_short` (DB-on-tmp + mocked LLM, no boot). The golden playthrough → `tier_medium` if driven through the TestClient WS (mirrors `test_scenario.py`); a direct-`execute_command` variant may sit in `tier_short`. Per-test budgets are the contract (short ≤2s, medium ≤10s).

**zat.env practices.** Land as small, individually-tested, committable increments (suggested order: `valid_iobj_kinds` + `give`/`use` + gate; `open`/`read` + state-aware examine; `spawn_object` verbs; loader/validator extension; author `worlds/clockmakers-loft.json`; parser fast-path; UI two-step + `verb_bar` payload; make canonical + docs/version). Write paired tests in the same increment; keep everything GPU/LLM/key-free via mocks in `short`/`medium`. Prefer proxy tests (deterministic state assertions) over critic gates. The deterministic playthrough is the durable regression guard — a future change that breaks the loop fails `bin/game test medium`.

**Out of scope (kept targeted).** The broader LLM-effect-target safety refactor (the new verbs are deterministic and scope-validated by construction, so they don't widen that gap); persistence-model/event-sourcing clarification; GPU arbiter metrics; multi-client load harness; player-authored skills/verbs; multi-level prototype chains; UI visual polish beyond functional two-step staging; a headless-browser test (frontend stays source-scan + one manual glance).
