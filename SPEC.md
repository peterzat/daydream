## Spec — 2026-07-01 — Reading Room UI theme + a durable design language (DESIGN.md)

**Goal:** Re-theme the live game UI (`web/`) into the chosen "Reading Room" storybook direction — wired to real `state_snapshot` data, preserving every existing interaction — and establish a durable UI design-language artifact (`DESIGN.md` + single-source CSS tokens + a drift-guard test) so all future daydream UI stays coherent and aligned with `WHIMSY.md`.

### Acceptance Criteria

- [ ] **The live SPA renders as the Reading Room storybook, fed by live data.** Loading the running game presents the storybook page: a matted, edge-feathered **chapter plate** holding the room image, a hand-lettered **title + folio subtitle** beneath it, the room narration as **drop-cap prose** on a paper-page surface, a right-margin **marginalia column** (`you` / `here with you` / `you carry`, plus an errand group when an objective is derivable), an **ink-tab "what you might do" verb ribbon**, and a **compass "ways from here"** exits footer. Every region is populated from the real snapshot (`snap.room`/`self`/`toons`/`items`/`inventory`/`verb_bar`/`exits`/`events`), not placeholders; the running event log (say/narrate/move) renders as continuing prose (newest last). Verifiable by a source-scan test of the new DOM/CSS plus a live visual review against `docs/mockups/01-reading-room/`.

- [ ] **No runtime interaction regresses through the new theme, and ids are never shown.** Clicking an in-prose affordance or a marginalia object stages/acts via the existing path; staging a verb dims non-applicable targets; a targeted examine/read reveals a ledger-style **detail inset**; two-step `give`/`use` still sends both dobj+iobj ids; exits navigate via the compass (`go <dir>`); the slots picker (`needs_toon`), the connection/hot-swap overlays, the `painting…`→`room_image_ready` image swap, and the redeploy auto-reload all still function; every actionable object stays reachable via the marginalia even when not present as an in-prose word; no object/toon id appears in any player-visible text. The existing frontend source-scan tests (`tests/test_frontend.py` pattern) are updated to the new DOM and pass, and existing `tier_short`/`tier_medium` suites stay green (runtime contracts unchanged).

- [ ] **The keepsakes backpack opens as a foldout over live inventory.** The backpack control opens a foldout/overlay rendering the two-page "Keepsakes" spread from `snap.inventory` — each carried item a specimen card (name, description, provenance/tag) plus empty collection slots — and closes back to the page, replacing the old "print `inventory` to chat" behavior. With an empty inventory it shows the empty-collection state without error. Verifiable by source-scan + a live open/close over a real inventory.

- [ ] **The storybook collapses to a single readable column on narrow screens.** At a phone-width viewport the layout reflows to one column (marginalia below the prose; ribbon and compass wrap; chapter plate scales) with no horizontal scroll, clipping, or overlapping regions, and stays usable for reading and clicking. Verifiable by a narrow-viewport render/manual check.

- [ ] **`DESIGN.md` is the durable UI bible and is wired in like WHIMSY.md.** A root `DESIGN.md` captures daydream's UI design language (UI anchors; color system; typography; layout & surface; component vocabulary; interaction patterns; banned patterns; a drift-check procedure), states its relationship to `WHIMSY.md` (WHIMSY owns world tone/voice/image; DESIGN owns the interface), and points to `docs/mockups/01-reading-room/` as the canonical reference. `README.md` and the **project** `CLAUDE.md` reference `DESIGN.md` ("read before touching `web/` UI or authoring component CSS"), matching how they reference `WHIMSY.md`. Verifiable by reading the file's sections and grepping both docs for the link.

- [ ] **Design tokens are a single source of truth, guarded by a tier_short drift test.** The palette + typography (+ radius/shadow) tokens live once as CSS custom properties under `web/assets/`; `DESIGN.md` carries a machine-extractable table mirroring them; a `tier_short` test extracts both and fails on any divergence (mirroring `tests/drift/test_drift_constants.py::test_whimsy_prompt_suffix_matches_code_constant`, with a clear "edit either side; DESIGN.md is the durable source" message). The token set is reconciled with the mockup (the live `--warm` #c8a06e standardized as `--amber`; a quest accent, and `--table`/`--parch`, and `--serif`/`--sans`/`--hand` added) with no dangling references left in the CSS. Verifiable: the test passes when synced and fails on an intentional one-sided edit, and runs under `bin/game test short`.

- [ ] **Local-only is preserved: the display font is self-hosted, no external assets.** The hand-lettered display face is bundled under `web/assets/fonts/` and loaded via `@font-face` (no Google Fonts / CDN link); `web/` references no third-party runtime asset (grep for `googleapis`/external `http(s)://` in asset links returns nothing), and the design degrades gracefully to system serif/sans stacks if the bundled font is unavailable. Consistent with the local-at-runtime generation policy.

- [ ] **Tiers stay green and version discipline is honored.** `bin/game test short` (including the new token drift test and the updated frontend tests) and `bin/game test medium` exit 0. The change is client-only (no `daydream/`/`worlds/`/DB/content mutation), so `daydream/version.py:WORLD_VERSION` is **not** bumped; new CSS/JS/font ride the existing `?v=<sha>` + `no-store` asset path (`daydream/api/nocache.py`) so a redeploy serves fresh assets.

### Context

**Origin.** Adopted from the user-approved plan `~/.claude/plans/i-want-you-to-squishy-fern.md` (read it for the full component-by-component live-data mapping and the four-direction exploration behind the choice). Prompted by: Peter reviewed four committed UI-direction mockups under `docs/mockups/` (Reading Room / Diorama / Companion Desk / Dream Pocket, each rendering the real Stopped Clock room) and chose **The Reading Room** with its keepsakes backpack. This turn makes it the live UI and captures the design language durably.

**Key facts the implementer needs.**
- **The render logic is largely style-agnostic.** `web/assets/main.js` `renderSnapshot()`/`renderEvent()`/`linkifyEntities()`/verb staging (`toggleStagedVerb`/`verbSpecs`/`applyVerbGating`/`onObjectClick`)/`sendCommand()`/WS reconnect+`?since=` replay/slots picker/overlays/redeploy-reload all stay. Net-new JS is scoped: bind the marginalia, add the detail (ledger) inset on staged examine/read, and add the backpack **foldout** (replacing `#backpack-toggle`'s current "send `inventory` → chat"). Most of the work is HTML restructure + CSS.
- **Token reconciliation.** Live tokens are in `web/assets/style.css:4` and use `--warm` (#c8a06e); the mockup + DESIGN standard is `--amber` (same value) — rename and update its few uses; add the mockup's extra tokens (`--table`, `--parch`, a quest accent #a97b3e, and `--serif`/`--sans`/`--hand`). The mockup's full token set + type scale is in `docs/mockups/01-reading-room/index.html`.
- **Local-only ethos (load-bearing).** The mockup used a Google-Fonts CDN `<link>`; the live game must NOT — self-host the `.woff2`. No cloud calls at runtime, period.
- **Client-only change.** No server/world/DB touch → **no `WORLD_VERSION` bump**; assets are already `?v=<sha>`-stamped + `no-store`.
- **Open items resolved (conservatively, to keep this client-only):** (a) the `errand`/objective marginalia group is **best-effort client-side** and requires NO new server field — derive it from a carried quest-tagged item if present, otherwise omit the group gracefully; (b) the running **event log stays**, styled as prose (not just the latest beat); (c) chapter/page ordinals ("chapter one", "— 1 —") are decorative unless trivially bindable (e.g. a visit count).
- **Drift-test model.** Mirror `tests/drift/test_drift_constants.py::test_whimsy_prompt_suffix_matches_code_constant` (also referenced in CLAUDE.md as `tests/test_whimsy_prompt_suffix.py`): extract the `DESIGN.md` token table, extract the CSS `:root` custom properties, whitespace-normalize, assert equality. Must run in `tier_short` (the pre-commit gate).

**Code anchors (verify before editing).** `web/index.html` (DOM regions), `web/assets/style.css` (`:root` at line 4; all component styles), `web/assets/main.js` (the functions above + `#backpack-toggle`, `#slots-panel`, `setRoomBackground`, `objectChip`). Tests: `tests/test_frontend.py` (source-scan posture — no headless browser), the new token drift test (extend `tests/drift/test_drift_constants.py` or add `tests/drift/test_design_tokens.py`). Asset stamping: `daydream/api/nocache.py`. Reference (unchanged): `docs/mockups/01-reading-room/{index,backpack}.html`, `docs/mockups/assets/scenes/*.png`. Durable model to imitate: `WHIMSY.md` + its cross-refs in `README.md`/`CLAUDE.md`.

**Tier placement.** Token drift test + updated frontend source-scan → `tier_short`. The live visual + responsive review is the one agent/manual glance — the keyless aesthetic gate (the agent Reads the running UI / a screenshot and grades it against `DESIGN.md` + the Reading Room reference), per the generation policy; escalate to `qpeek`/browser for a human eye if wanted. Existing `short`/`medium` stay green.

**zat.env practices.** Land as small, individually-tested, committable increments (suggested order: `DESIGN.md` + reconciled tokens + drift test **first**, as the source of truth; then the SPA structural retheme; then the backpack foldout; then responsive; then self-host the font + cross-refs + doc updates). Write paired tests in the same increment; keep everything GPU/LLM/key-free. Prefer source-scan proxy tests + the one agent visual glance over a headless-browser dependency (consistent with the project's frontend-test posture). Change only what the retheme needs; do not refactor unrelated runtime code.

**Out of scope.** The other three directions (Diorama / Companion Desk / Dream Pocket, kept as reference mockups); any gameplay/world/verb/server change (`daydream/*`, `worlds/*`); a server-side objective/quest model; bespoke per-item keepsake illustrations (generic parameterized SVGs in v1); a headless-browser test.
