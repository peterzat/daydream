## Spec — 2026-07-02 — Dreamseeds: grow the world from the inside (v0.5.0)

**Goal:** Build the README's headline "semi-procedural" moment: a quest-earned **dreamseed** a player `plant`s, answering one in-character question with a short phrase, to grow ONE new persistent room — composed by the local LLM inside the seed's Opus-authored boundaries, linked by a real exit, visible to every player forever after. This is the first consumer of the `spawn_room`/`link_exit` effect vocabulary (documented-not-built in `daydream/skills/effects.py` today) and the first end-to-end proof of the project's pre-bake-plus-local-polish thesis on world growth.

### Acceptance Criteria

- [x] **The planted-seed loop works end to end.** With a carried dreamseed and a short vision phrase, `plant` makes exactly ONE local-LLM call (assertable via a mocked-LLM spy) and produces, atomically from the player's view: a new persistent `kind='room'` object whose title/seed/description come from the validated LLM output; a new exit on the planting room in an engine-picked free compass direction (first free of north/east/south/west/up/down; the LLM never sees directions or ids) plus the reverse exit on the grown room; 0–2 spawned objects resting in the grown room; the seed consumed (state `spent`, `plant` no longer offered on it, moved into the grown room as a husk whose examine reads an authored husk line); a final in-character narrate naming the direction and title; and room provenance recording the seed id, planter id, verbatim phrase, and timestamp (`generated_by="plant:<seed-id>"` + a structured `grown` property). Verifiable by a deterministic golden-playthrough extension in `tests/test_quest_playthrough.py` (mocked LLM, zero real GPU).

- [x] **Growth is bounded and validated before any mutation.** The LLM output must pass, in order: the refusal escape hatch (`{"refused": true, "reason"}` narrates the reason); a strict schema (title 3–40 chars/1–5 words, room_seed 30–300, description 60–500, 0–2 objects with name 3–30 + seed 15–200; more than 2 objects rejects, not truncates); the WHIMSY banlist over every text field; and an anti-copy check against the seed's authored exemplars. The player's phrase is length-capped (~120 chars), input-banlist-checked, and enters the prompt only inside role-separator wrapping. `DAYDREAM_GROWTH_MAX_ROOMS` (default 12, counting only grown rooms in the world) refuses further growth in-character, checked both before the LLM call and again at commit. Any rejection mutates NOTHING. Verifiable by per-rule unit tests in a new `tests/test_growth.py`.

- [x] **Every failure path preserves the seed.** vLLM down → the foggy outage line; empty vision → the seed's authored in-character question (the typed two-turn path, no session state); schema/banlist/anti-copy failure → a soft in-character line; refusal → the reason; cap reached or all six directions taken → in-character refusal. In every case the seed stays carried, unspent, and plantable, and no room, exit, or object exists afterward. Concurrency is pinned: state changes during the LLM await (seed dropped/taken, cap reached, preferred direction occupied by a rival plant) are re-checked in a post-LLM synchronous commit block — no half-linked exits, no double rooms, no spent-but-roomless seeds. Verifiable by failure-path and race unit tests (mock the LLM to mutate state mid-call).

- [x] **`spawn_room` and `link_exit` join the allowlisted effect vocabulary correctly.** Both implemented in `daydream/skills/effects.py` and emittable ONLY by verbs that declare them: `plant` is the sole consumer, and an NPC dialogue / data skill / other verb attempting either kind is rejected exactly like an unknown kind (no mutation, fallback narrate). `spawn_room` writes the room property shape the existing `rooms.Room` reader consumes, so a grown room is first-class with zero reader changes (description on entry, exits, image-gen seed, `go to <place>` by name). `link_exit` validates both rooms and both directions before writing either side (all-or-nothing, never a one-way link). `destroy_object` remains documented-not-built. Verifiable by effects unit tests including allowlist-rejection cases.

- [x] **Both producers and the SPA drive `plant`, and the world updates live.** Click path: Plant appears on the verb bar, staging it dims everything but growth items, clicking the seed prompts for the vision (mirroring Talk's prompt) and sends one structured command frame. Typed path: bare `plant` answers "Plant what?" with zero LLM calls; "plant the dreamseed ..." grounds through the parser with the vision carried into args. After growth, the planter's snapshot refreshes in place so the new exit appears without reconnecting; `go <direction>` enters the grown room and shows its full stored description; the room's watercolor background renders lazily on first entry through the existing persistent-image path with zero new image-gen code. No object/toon/room id appears in any player-visible text. Verifiable by a tier_medium WS integration test (`test_ws_grow.py` pattern) plus a frontend source-scan assertion.

- [x] **The seed is quest-earned and the world versions correctly.** In `worlds/clockmakers-loft.json`, opening the clock case now reveals BOTH the warm brass cog AND the dreamseed (the envelope's `contains` accepts a single object or a list; the loader validates a `growth` block fail-loudly: question, theme 1–8, palette, motifs 0–8, exemplars 1–3 of title/seed/description, optional skeletons 0–3, unknown keys tolerated). The dreamseed declares `plant` via per-object verbs and carries Opus-authored growth boundaries with 2–3 exemplar rooms in the loft's voice. The existing zero-LLM golden quest playthrough still passes unchanged. `WORLD_VERSION` bumps 1.1 → 1.2 and the boot gate treats a live 1.1 world per MINOR rules (warn, not refuse). Verifiable by loader/world-integrity tests + the unchanged quest golden.

This is the increment the substrate was built for: README "Semi-procedural gaming" describes exactly this loop, and `effects.py`'s docstring reserves exactly this vocabulary.

**Key design decisions the implementer inherits (do not relitigate):**
- New module `daydream/growth.py` owns the pipeline: gates → one `acompletion_json` call (temp 0, ~450 max tokens, ~30 s timeout; arbiter is internal to the client) → validate → post-LLM synchronous commit block (re-check gates, pick unique slug + `r-<slug>` id, dispatch ONE ordered effect batch). Effect order: `spawn_room` → `link_exit` → spawn_object(s) → seed `set_property state/examined_text/verbs` → `move_object` husk → `narrate`.
- `plant` is a closed-set VerbSpec (needs_dobj, thing, `free_text=True`, on the bar) granted per-object by the seed's `verbs: ["plant"]` — the clock-case/case-key idiom. Handler is a thin delegate (lazy import, like `_handle_talk`). A non-growth thing refuses via the existing `verbs_for` gate; a growth-less item with `plant` gets an in-character "nothing in it wants to grow."
- `_handle_open` normalizes `contains` to a list and forwards each entry's optional `properties` into the spawn effect (`spawn_object` gains a properties passthrough; computed keys win). Existing reveal dedup (name+location+provenance) is unchanged.
- WS: `_EFFECT_MUTATION_KINDS` gains `room_grown` + `exit_linked` (narrow new kinds; do NOT add `property_set`, whose blast radius is every mood write and examine-cache). The batch's other events already trigger refreshes; multiple refresh snapshots per plant are acceptable and idempotent.
- Parser: zero code changes expected — `plant` enters the vocabulary from `verbs.VERBS`; `free_text=True` defers "plant the seed to X" arg extraction to the grounding LLM. Verify, don't assume.
- The growth prompt is an engine constant (dream-gardener system prompt + user block of seed boundaries/exemplars/current-room context/wrapped phrase). Exemplars are authored into the world envelope by the agent in-session (the pre-bake; no API key). `skeletons` is validated from day one so a rung-(b) flip is prompt+validator only.
- Suggested increment order (each committable, test-green): (1) effects vocabulary, (2) loader + open reveal, (3) growth core, (4) verb wired + WS, (5) authored seed + WORLD_VERSION + golden extension, (6) SPA, (7) tier_long probe + ratification, (8) docs roll-forward + world reset + live playthrough.
