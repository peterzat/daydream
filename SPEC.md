## Spec — 2026-06-30 — playtest polish: playable first-entry, clear scene + inventory, NL/voice/forge fixes

**Goal:** Close the gap a real playtest exposed: the object/verb game is broken on first entry and unclear to play. Make it actually playable and legible from the first connect — a picker-first entry flow, a scene that says who you are / who's here / what's on the ground / what you carry, a real inventory affordance, applicable-only verbs, and the natural-language, narration, NPC-voice, and forge-image fixes the playtest surfaced.

### Acceptance Criteria

*Entry flow (critical)*

- [ ] **A fresh/unclaimed session lands on the character picker, never a phantom toon.** A WS session with no claimed/controllable toon (fresh connect, after "leave the dream", or whose toon was kicked/deleted) receives a `needs_toon` frame and the SPA shows the picker; it never resolves to a nonexistent or arbitrary toon, so no input ever silently no-ops to the "you think to yourself" fallback for lack of an actor. Claiming or creating a toon enters the dream and routes input to it. Verifiable via TestClient against a `world load`ed world (uuid'd toon ids, no literal `t-wren`): connect-without-claim → `needs_toon`; after claim → "go north" moves. The legacy `t-wren` fallback is removed and the seeded-world WS tests that relied on it claim/create first.
- [ ] **No raw ids appear in anything the player reads.** No object/toon id (`t-iris-ed7…`, `o-…`, `r-…`) appears in any narration, dialogue, say-attribution, or rendered UI label (data attributes may still carry ids). Verifiable: talk / say / examine / scene tests assert names (not ids) in visible text, including a regression test for the talk-attribution path that surfaced `iris-ed7…`.

*UI clarity + inventory*

- [ ] **The scene reads unambiguously.** The SPA distinctly labels and separates: WHO YOU ARE (the controlled toon, shown explicitly), WHO ELSE IS HERE (co-located toons), WHAT'S ON THE GROUND (room things), and WHAT YOU'RE CARRYING (inventory). Verifiable via a frontend markup test (labelled self / here / ground / carrying regions) + the snapshot carrying the controlled toon's identity.
- [ ] **Inventory is a first-class affordance.** An `inventory` text command AND a backpack-style UI control both surface the toon's carried things (things located on it), with a clear empty-inventory message, built on the existing objects containment model (no new schema). Verifiable: a mocked-LLM TestClient test (`inventory` lists carried things; says so when empty) + a frontend markup test for the backpack control.
- [ ] **Verbs are offered only where they apply.** The verb bar enables a verb for an object only when applicable (Talk → toons; Take/Drop → things; Examine → toons + things), so staging Talk then clicking a thing is impossible and the client never prompts for talk text on a non-toon; an inapplicable verb+target that still reaches the server is refused with narration and no state change. Verifiable: a frontend markup/logic test for per-object verb gating + the existing server-side refusal test.

*Natural-language + narration fixes*

- [ ] **Navigate by place name.** From a room, "go to \<place>" (a name, alias, or slug of a room reachable by one exit of the current room) moves you there; a place that is not an adjacent exit, or is unknown, is refused with narration and no move (no multi-hop pathfinding this turn). Verifiable GPU-free via a deterministic place-name → exit-direction resolution test (adjacent → move; non-adjacent / unknown → refusal).
- [ ] **"look at \<object>" examines it.** "look at the lantern" / "look at rook" resolves to examine(that in-scope object), not a room look. Verifiable GPU-free via a deterministic fast-path test ("look at \<in-scope name>" → examine).
- [ ] **Player-action narration is second person.** Room-affordance skills (stoke / tend) and any player-action narration address the player as "you," never "the visitor" (third person). Verifiable: the authored affordance prompt templates + the data-skill system message instruct second person (a static-scan test asserts no "a visitor" / "the visitor" framing in the authored prompts) + a one-time eyeball that runtime output reads "you."
- [ ] **Verb narration is clean and specific.** Examine narration never double-punctuates (detail already ending in terminal punctuation does not yield ".."); and a verb targeting a *named-but-absent* object ("take the moon") yields a "you don't see a/the \<thing> here"-style line, distinct from the bare "Take what?" used when no target was named at all. Verifiable: a verb unit test (detail ending in "." → single terminal period) + a mocked-LLM test (named-absent target → "don't see" narration).

*Connection states*

- [ ] **Calm SPA states when the dream is interrupted.** When the WS connection drops and reconnect attempts keep failing (the server is down — e.g. during a `bin/game down`/`up` cycle), the SPA shows a single calm, on-theme "the dream is sleeping…" state — not a growing pile of disconnect/error lines — keeps quietly retrying on a sane cadence (gentle, capped backoff), and seamlessly restores the live view once the server returns, with no manual refresh, so a browser tab left open across a restart degrades gracefully and recovers on its own. The same overlay covers the live world-swap `WORLD_CHANGED` signal as a brief "the dream shifts…" beat into the fresh snapshot (the named follow-up from world-hot-swap). Verifiable via a frontend markup test (the sleeping/transition overlay element + the reconnect retry/backoff logic + the `world_changed` handler) + a one-time browser eyeball across a real down/up cycle and a swap.

*NPC voice + forge image (qualitative, flagged manual)*

- [ ] **NPC dialogue reads in-character and terse.** The per-NPC `talk` dialogue prompts (Rook laconic; Iris bookish/soft; Bram gentle) are tightened toward terse, sensory, in-voice output that holds on the local Qwen 2.5 7B; the authored prompts encode the per-NPC voice constraints (testable presence), and a one-time browser/qpeek eyeball confirms the voices read distinct and not florid. If a target voice will not hold on the 7B, that is flagged in-session and pre-baked / simplified per the generation policy rather than shipped flat.
- [ ] **The forge looks like a forge.** The `r-forge` seed / image prompt renders a working forge (anvil, bellows, tools), not a domestic hearth, in the watercolor WHIMSY tone; verified by a one-time browser/qpeek eyeball (flagged manual).

*World feel*

- [ ] **Ambient NPC drift is witnessed, not hidden.** Drift is no longer suppressed in a room a human occupies; a co-located NPC's ambient body-language drift (e.g. "Rook brushes soot from the anvil and lets it settle") is shown to the present player as a quiet "the world is alive around you" beat. The cadence is retuned for this: with a player present, ambient NPC action is witnessed periodically — a minutes-scale interval, not the prior 30-minute busy cadence (which existed precisely to keep drift away from connected players), and never every tick — so it reads as alive, not spammy or intrusive. Verifiable: a drift tick with a present human + a co-located NPC emits that NPC's narrate to the room (inverting today's `test_tick_suppresses_drift_in_occupied_room` behavior); the present-player cadence constant is retuned to a minutes-scale value; a one-time eyeball confirms the feel. The mood-nudge-per-tick behavior is unchanged. README + CLAUDE.md drift sections updated to describe witnessed drift rather than occupancy suppression.

*Cross-cutting*

- [ ] **Tiers green; a gameplay-scenario test; docs + backlog rolled forward.** `bin/game test short` and `bin/game test medium` exit 0 with paired GPU/LLM-free tests (mocked LLM) for every non-qualitative criterion above; a new scripted multi-step gameplay-scenario test exercises the playable flow end to end (connect → picker → claim → look → take → go-to-place → talk → spawn → examine → inventory). README + CLAUDE.md updated for the picker-first entry, the inventory affordance, verb-applicability, and the connection-states overlay (resolving CLAUDE.md's "the dream shifts" swap-transition follow-up); `BACKLOG.md` swept (the four 2026-06-29/30 DONE entries removed; `gameplay-scenario-tests` closed; new deferred items recorded — toon-delete should DROP carried items to the room rather than destroy them, and dialogue-latency / parser-throughput tuning).

### Context

**Origin.** Driven by the 2026-06-30 browser playtest + an agent WS playtest of the just-shipped object/verb world (SPEC "objects + local LLMs", 19/19). Operator chose the BROAD scope (all of the above in one turn) and the **picker-first** entry-flow decision (over auto-claim / auto-create).

**The critical bug, precisely.** `daydream/api/ws.py` resolves an unclaimed session to `LEGACY_TOON_ID = "t-wren"`. A `world load`ed world gives toons uuid'd ids (`t-wren-<uuid>`), so `t-wren` does not exist; `parser.parse` returns `none` for a missing actor and every input (including the exit buttons and verb/object clicks, which the SPA sends as input/command frames) falls through to "you think to yourself." The fix is to route a no-controllable-toon session to the existing `needs_toon` / picker path (today only taken when the session is marked `left`). Removing the fallback updates the WS tests that connect unclaimed and expect to act as `t-wren` (`test_ws*`, e.g. `test_ws_unclaimed_session_falls_back_to_t_wren`).

**Code anchors (verify before editing).** Entry flow + ids: `daydream/api/ws.py` (`_resolve_controlled_toon_id`, the `needs_toon` branch, `_handle_input`, say-attribution), `web/assets/main.js` (`enterPicker`, `renderSnapshot`). Parser/verbs: `daydream/parser.py` (deterministic fast-path for "go to \<place>" and "look at \<name>"; pass a *named-but-unresolved* dobj through), `daydream/verbs.py` (examine punctuation in `_examine_line`; the "you don't see" vs "Take what?" split; per-kind verb applicability already in `valid_dobj_kinds` / `verbs_for`). UI: `web/index.html` + `web/assets/main.js` + `style.css` (labelled scene regions, the backpack control, client-side verb gating). Content/voice: `worlds/bunny.json` (per-NPC `dialogue.prompt_template`, the `stoke`/`tend` affordance prompts → second person, the `r-forge` seed), `daydream/skills/data.py` (the dispatcher system message → second person). Inventory: `daydream/objects.py` `contents(toon, 'thing')` already is the model; add the verb (a `verbs.py` handler or a parser fast-path) + the UI control.

**Generation policy (load-bearing — CLAUDE.md).** Runtime is local-only; NPC voice is prompt/authoring work within the Qwen 2.5 7B budget, not a model swap. If a voice won't hold on the 7B, say so here and pre-bake/cache/simplify rather than ship flat. Tests stay GPU/LLM-free by mocking `daydream.llm.client`; the voice + forge-image criteria are the only qualitative ones (eyeball once, flag as manual, like prior specs' browser checks).

**zat.env practices.** Land as small, individually-tested, committable increments (suggested order: entry-flow/picker fix first — it unblocks everything — then UI clarity + inventory, then the NL/narration fixes, then voice + forge, then the scenario test + docs). Write paired tests in the same increment; never stack untested changes; run `bin/game test short` between increments. Acceptance criteria are the contract the pre-push `/codereview` checks for spec alignment.

**Connection states (the calm-interrupt criterion).** Purpose: a browser tab the operator leaves open survives `bin/game down`/`up` cycles with a pleasing "sleeping" visual + auto-recovery instead of WebSocket-fail spam. The mechanism to improve is already there — `web/assets/main.js` `ws.onclose` retries `connect(true)` every 1.5 s but emits a "(disconnected; reconnecting)" line each time and has no overlay. This criterion folds in the **world-hot-swap "the dream shifts" SPA-transition follow-up** (CLAUDE.md, world-hot-swap "remaining polish"): the same overlay handles the `WORLD_CHANGED` re-snapshot. So deleting the shipped world-hot-swap backlog entry at turn close loses nothing.

**Backlog.** Revives `gameplay-scenario-tests` (the scenario test is criterion-final) and absorbs world-hot-swap's "dream shifts" follow-up (above). Explicitly DEFERS (note, don't build): `object-lifecycle-clutter-gc` + a new toon-delete-drops-items item, `parser-latency-and-throughput-tuning` (the felt dialogue latency: talk = parse + dialogue serialized with image-gen), `player-touch-object-promotion`, `two-object-verbs`, user-authored world-building.

**Out of scope.** Deeper inventory mechanics (capacity / nested containers / weight); LLM latency/throughput tuning; the deferred backlog depth; any model swap.
